## 5. Database Query Patterns

### 5.1 Hot Queries (>100 executions/day)

**Q1: Navigation Badge Count**
```sql
-- Runs on EVERY page load
SELECT COUNT(*) FROM callback_requests
WHERE company_id = ?
  AND status = 'pending';

-- Index used: idx_company_status_priority_expires
-- Performance: ~2ms
-- Optimization: Cache with 5-minute TTL âœ…
```

**Q2: Table Listing (Filament)**
```sql
-- Runs on callback index page
SELECT * FROM callback_requests
WHERE company_id = ?
ORDER BY created_at DESC
LIMIT 15 OFFSET 0;

-- Index used: idx_company_created
-- Performance: ~5ms
-- Optimization: Eager load relationships âœ…
```

**Q3: Overdue Check (Scheduled)**
```sql
-- Runs hourly
SELECT * FROM callback_requests
WHERE expires_at < NOW()
  AND status NOT IN ('completed', 'expired', 'cancelled');

-- Index used: idx_callback_overdue
-- Performance: ~8ms (with 1000+ records)
-- Optimization: Good index coverage âœ…
```

### 5.2 Complex Queries (Require Optimization)

**Q4: Staff Load (Auto-Assignment)**
```sql
-- Runs on every callback creation
SELECT staff.*, COUNT(callback_requests.id) as callback_count
FROM staff
LEFT JOIN callback_requests
  ON callback_requests.assigned_to = staff.id
  AND callback_requests.status IN ('pending', 'assigned')
WHERE staff.branch_id = ?
  AND staff.is_active = true
GROUP BY staff.id
ORDER BY callback_count ASC
LIMIT 1;

-- Performance: ~15ms
-- Optimization: Add composite index on (assigned_to, status)
```

**Q5: SLA Compliance Rate (Dashboard)**
```sql
-- Runs on dashboard load
SELECT
  priority,
  COUNT(*) as total,
  SUM(CASE WHEN completed_at <= expires_at THEN 1 ELSE 0 END) as on_time,
  ROUND(100.0 * SUM(CASE WHEN completed_at <= expires_at THEN 1 ELSE 0 END) / COUNT(*), 2) as compliance_rate
FROM callback_requests
WHERE company_id = ?
  AND status = 'completed'
  AND created_at >= NOW() - INTERVAL '24 hours'
GROUP BY priority;

-- Performance: ~25ms
-- Optimization: Cache with 1-hour TTL (recommended)
```
