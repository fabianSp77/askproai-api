🏦 Wie ein Tresor für Ihre Daten

Stellen Sie sich vor: Ihr Backup-System ist wie ein hochmoderner Bank-Tresor, der dreimal täglich automatisch eine perfekte Kopie Ihrer gesamten Firma erstellt. Nicht nur einmal am Tag - sondern morgens, mittags und abends. Wenn etwas schiefgeht, können Sie jederzeit zurück zu einem dieser Zeitpunkte springen.

Noch besser: Zwischen diesen großen Kopien wird zusätzlich jede einzelne Änderung protokolliert (wie eine lückenlose Video-Aufnahme). So können Sie nicht nur zu den 3 Backup-Zeitpunkten zurück, sondern zu jedem beliebigen Moment - auf die Sekunde genau!

⏮️ Point-in-Time Recovery (PITR)

🎬 Wie eine Video-Zeitmaschine

Normale Backups sind wie Fotos: Sie haben Momentaufnahmen von 07:00, 13:00 und 19:00 Uhr.

PITR ist wie ein Video: Sie können zu jedem Zeitpunkt zwischen den Fotos springen. Ein Fehler passierte um 14:37 Uhr? Kein Problem - Sie können exakt zu 14:36 Uhr zurück!

Wie funktioniert das?

  1. Basis-Backup wiederherstellen
    Wir nehmen das letzte Backup vor dem Problem (z.B. 13:00 Uhr)
  2. Änderungen nachspielen
    Das System spielt alle protokollierten Änderungen von 13:00 bis zu Ihrem Wunsch-Zeitpunkt (z.B. 14:36) nach
  3. Exakte Wiederherstellung
    Ihre Datenbank ist jetzt exakt im Zustand von 14:36 Uhr - auf die Sekunde genau!
📌 Beispiel: Um 14:37 Uhr wurde versehentlich eine wichtige Kundenliste gelöscht. Mit PITR können Sie zu 14:36 Uhr zurück - eine Minute bevor es passierte. Die Kundenliste ist wieder da! ✨

🛡️ Sicherheits-Features

🔐
Verschlüsselte Übertragung
Alle Backups werden über SSH übertragen - wie ein Panzerwagen für Ihre Daten.
✓
Integritätsprüfung
Jedes Backup bekommt einen digitalen Fingerabdruck (SHA256). So erkennen wir sofort, wenn etwas manipuliert wurde.
🚨
Wenn ein Backup plötzlich 50% größer/kleiner ist als normal, schlagen wir Alarm - könnte ein Fehler sein!
Größen-Anomalie-Erkennung
🔒
Kein gleichzeitiges Backup
Lockfile verhindert, dass zwei Backups gleichzeitig laufen - könnte zu Chaos führen!
💾
Lokale + Externe Kopie
Die letzten 3 Backups bleiben auf dem Server (schneller Zugriff), alle Backups auf der NAS (externe Sicherheit).
📧
E-Mail bei jedem Backup
Sie bekommen nach jedem Backup eine E-Mail - egal ob erfolgreich oder fehlgeschlagen. Transparenz garantiert!

🗄️ Wo werden Backups gespeichert?

📍 Synology NAS (Hauptspeicher)

Standort: /volume1/homes/FSAdmin/Backup/Server AskProAI/

Struktur:

  • daily/ - Tägliche Backups (14 Tage aufbewahrt)
  • biweekly/ - Zweiwöchentliche Snapshots (6 Monate aufbewahrt)
  • binlogs/ - Änderungs-Protokolle für PITR

💻 Lokaler Server (Schnellzugriff)

Standort: /var/backups/askproai/

Inhalt: Die letzten 3 Backups - für schnelle Wiederherstellung ohne NAS-Zugriff

Aufbewahrungsfristen

🔧 Wiederherstellung: Was passiert im Notfall?

⚠️ Wichtig: Immer IT kontaktieren!

Eine Wiederherstellung sollte nie ohne IT-Expertise durchgeführt werden. Dieser Abschnitt erklärt den Prozess für Ihr Verständnis, nicht zur Selbst-Ausführung.

Szenarien & Lösungen

1️⃣ Datei wurde versehentlich gelöscht (vor 2 Stunden)

Lösung: Point-in-Time Recovery

  1. Letztes Backup wiederherstellen (z.B. 13:00 Uhr)
  2. Änderungsprotokolle bis kurz vor Löschung nachspielen
  3. Datei ist zurück! ✨

Dauer: ~15 Minuten

2️⃣ Kompletter Server-Crash

Lösung: Vollständige System-Wiederherstellung

  1. Neuen Server aufsetzen
  2. Letztes Backup von NAS laden
  3. Datenbank + Dateien + Konfiguration wiederherstellen
  4. System läuft wieder! ✨

Dauer: ~2 Stunden

3️⃣ Daten von vor 3 Tagen benötigt

Lösung: Backup vom entsprechenden Tag

  1. Backup vom gewünschten Tag auf NAS finden
  2. Nur die benötigten Daten extrahieren
  3. Daten bereitstellen - System läuft weiter! ✨

Dauer: ~30 Minuten
